ai

a lisp where every value is a total function of one argument — and every value has a measure, so truth is just the sign of that measure. stackless, overflow-safe, fully generic: lisp, apl and ml grown over portable public-domain C. the terminal below runs the very image that boots on bare metal, compiled to wasm and evaluating in your browser.

start here

1 + 2 * 3 (i love you) (map (* 2) (jot 10)) @(1 2 3) * @(10 20 30) (L +"love" <>'(1 2 3)) (: (fact n) … (fact 30)) .ev

(i love you) is 1: i applies as the identity count, an unbound name reads as nothing, and nothing applies as the constant — so absence absorbs and the answer is one. the terminal shows the receipt, ;; missing love: a miss is not an error here, it is a value that tells you its name. +"love" is its net — every value carries a measure, and + turned inward is the true sum. and .ev is no joke: it prints the compiler the compiler compiled itself into.

the shape of it

numbers are church numerals, so application is arithmetic. every quote below evaluates to 1 — true — in the terminal. click them.

1 = (0 5)0 is the constant; nothing applied to anything is one
5 = (1 5)1 is the identity
262144 = (2 3 4)(n x) = xⁿ; a list of numbers is a tower
12 = (3 (+ 1) 9)a number composes a function n times
-1 = i * icomplex is in the tower
(log -1) = i * pieuler, exact in the principal direction
i = (1 / 2) -1sqrt is a numeral; roots are principal on the nose
2 = <>'(1 2 3)the valence law: a glued operator is monadic — < is cap, > is cup, they compose
'(1 1 0 6) = (L !0 !!7 ?-3 $'(1 2 3))truth is the sign of a measure; ?x is the iverson bracket, $x the measure's charm
'(1 0 0) = (L $'(1) $'(0) $'(-2 1))the color law: $ keeps the green, floors the blue, clamps the red to nothing

something to play with

the triangular numbers, point-free — jot lays out 0 .. n-1, sat sums each prefix into its charm, and flip compose feeds the one into the other:

((flip compose jot (map (compose sat jot))) 9) → (0 0 1 3 6 10 15 21 28); the nth is the net of 0 .. n-1

fizzbuzz — a number applies its operand n times, so the whole loop is the number 15 applied to a step function:

run fizzbuzz the program is one application: (15 step 1)

build it yourself

make builds and tests the host binary; make repl is the shell; make test_all adds the freestanding kernel and the tool gates. the whole thing is portable C with zero dependencies — host, bare metal (x86_64 / aarch64 via limine), and the wasm on this page from one source.

the crew

ai is sailed by a small crew — ten little programs, each with one thing it tends, each written in ai over the same tiny core. they ride kship, the freestanding kernel (a ship in port). meet them:

ai
the pilot — the mind at the helm, the decide in perceive→decide→act.
siri
the synthesist — keeps the human words and the ai words saying the same thing.
sift
the garbage collector — tends the two-space litter box, keeps it clean.
bao
the shell — one editor and the condition system, a soft wrap around a chewy command.
aineko
the cat — a netcat clone (愛猫), bytes pumped both ways over a socket.
cook
the builder — make-in-ai, builds the host from scratch and passes the corpus.
tele
the eye — a telescope that scopes constellations: tensors and autograd.
zev
the reader — a parser-combinator library, tiny functions that compose.
wev
the weaver — a partial evaluator, hands back a smaller program from what's known.
kship
the ship — the freestanding kernel itself, the language running on bare metal.

each links to a short bio; the full agent briefs are the crew/*.md files in the repo.
