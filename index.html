ai

a lisp where every value is a total function of one argument. stackless, overflow-safe, generic to the bone — lisp, apl and haskell fused over portable public-domain C. the terminal below runs the real thing: the same image that boots on bare metal, compiled to wasm, evaluating in your browser. nothing is mocked.

enter evaluates a complete line (each line is one expression — infix works); an open paren continues to the next line; up and down walk history. errors print # condition data and the session keeps going.

start here

1 + 2 * 3 (i love you) (map (* 2) (jot 10)) @(1 2 3) * @(10 20 30) (L +"love" <>'(1 2 3)) (: (fact n) … (fact 30)) .ev

(i love you) is 1: i applies as the identity count, an unbound name reads as nothing, and nothing applies as the constant — the terminal shows the receipt, # missing love. +"love" is its net — every value here has a measure, and + turned inward is the true sum. and .ev is not a joke: it prints the compiler the compiler compiled itself into.

the shape of it

numbers are church numerals, so application is arithmetic. every quote below evaluates to 1 — true — in the terminal. click them.

1 = (0 5)0 is the constant; nothing applied to anything is one
5 = (1 5)1 is the identity
262144 = (2 3 4)(n x) = xⁿ; a list of numbers is a tower
12 = (3 (+ 1) 9)a number composes a function n times
-1 = i * icomplex is in the tower
(log -1) = i * pieuler, exact in the principal direction
i = (1 / 2) -1sqrt is a numeral; roots are principal on the nose
2 = <>'(1 2 3)the valence law: a glued operator is monadic — < is cap, > is cup, they compose
'(1 1 0 6) = (L !0 !!7 ?-3 $'(1 2 3))truth is the sign of a measure; ?x is the iverson bracket, $x the measure's charm

build it yourself

make builds and tests the host binary; make repl is the shell; make test_all adds the freestanding kernel and the tool gates. the whole thing is portable C with zero dependencies — host, bare metal (x86_64 / aarch64 via limine), and the wasm on this page from one source. the spec is CLAUDE.md; the readme is README.md.
