ai is a dynamically typed lisp superset with curried evaluation, fast generics, and optional infix/prefix notation. other facts include

  • every value is a theoretically a one argument total function of the global type
  • barring conditions such as out of memory or infinite loop, pretty much every expression has a value
  • most functions are defined generically, the goal is fully generic everywhere
  • alpha-equivalent lambdas compare equally: (\ x x) = (\ y y)
  • integers are considered equivalent to their church numerals: 1 = (\ x x)
  • infix/prefix notation (desugars to lisp)
  • integer, float and complex types plus shaped numeric arrays with broadcasting
  • mandatory co-op scheduling
  • macros, call/cc, optional static dependent type checking (in early development)
  • repl with native line editing
  • self-hosting threaded code interpreter with template JIT compiler (amd64/arm64 only)

glossary

i made up my own words for some things

application
the main vm operation: (f x) applies f to x. the numeric case is exponentiation.
band / the true-blue order
the cross-kind total order: () < mint < name < string < number < tray < chain < book < hot -- numbers folded into one band by net.
blue
net exactly 0: (0, (), "", '(1 2 3 -6)).
book / tablet
a mutable hash that is its own lookup function. the book refers to the global scope.
chain
the product kind (cons/pair). cap is car/hd, cup is cdr/tl, with the usual abbreviations. (btw i know pairs are not string diagrams, i just like the words :)
charm
a register-allocated fixed-width integer (fixnum)
color
the order-sign of a value's net: green, red, or blue.
constellation
any number -- a star or a galaxy.
currying
(f x y) = ((f x) y), (f) = f -- a multi-arg call is just application, left to right.
galaxy
a tray of stars (numeric array); folds into the number band by its net.
gem
a float -- the smooth kind between the charms.
the kind lattice
the runtime-reflective type lattice that indexes the dispatch tables, extending the generic ops to every kind.
green
net ≥ 0 -- what $ keeps; the naturals.
hot
an opaque external reference; applies as a constant.
kind
a value's type tag; the kinds form the dispatch lattice.
mint
a fresh nameless point: an identity and nothing else, materially empty.
net (+)
a value's content measure, a complex number; truth is its sign. a list nets its spine, text its charms.
nom / name
a named point (spelling + serial). a name is interned by the environment and pinned to one canonical object.
numeral
a charm as a Church numeral: (n x) iterates x n times. 1 is identity, 0 const-1.
operator
the global type: every value is a one-argument total function, so application is always defined.
point
a mint or a nom; the points floor the total order, below string.
red
net < 0 -- what $ clamps to nothing.
saturate ($)
retraction of the net measure onto green charms: $x = (max 0 (min max-charm (net x))). the truth value of x is !!$x.
star
a self-netting scalar: a number that is its own net.
string
a byte vector: indexes its bytes, nets its charm-sum.
tally
the count -- how many cells, charms, or keys -- versus the net's how-much.
tray
a shaped array, cells of any kind -- seated just above chain in the band. a shape lists its axis sizes, the rank counts them; indexing is row-major and out of bounds reads the default. typed per numeric tier (star/gem/twin) or holding objects; a numeric tray is a galaxy and nets its cells' sum. one cell demotes to the bare scalar, so a tray exists only at tally ≥ 2 -- save the empties, which stay to carry the reduction units.
twin
a complex: two gems (re, im), built and conjugated by ~.
unit / zero point
the global empty element: () + x = () * x = x. canonically a mint, therefore false. the face of absence a missing read returns.
