zev · the reader

tiny functions that compose a string into a tree.

zev is a parser-combinator library: a small, tested vocabulary for turning a string into a tree, built out of tiny functions that compose. a literal, a single char, a maximal run; ordered choice, sequence, repetition — each one is nothing on its own and everything once you stack them. that is the whole of the idea, and the rest is bookkeeping.

the idea is already proven aboard: cook's Makefile importer is built from exactly these (\ s y n) parsers, reading real GNU Makefiles. but cook keeps them locked inside one private body, reinvented the next time anyone needs to read a wire header or a command line. zev lifts that block out, generalizes it, and offers the whole crew one shared combinator vocabulary instead of N hand-rolled char loops.

Ze'ev is the wolf, and a wolf hunts in a pack — which is the shape of the thing exactly: a combinator alone does little, a pack of them reads a grammar. zev keeps a careful house too, exposing just one name so it can ride the same scope as everyone else without ever colliding.
