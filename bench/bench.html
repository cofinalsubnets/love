ai benchmarks — milliseconds per iteration

Lower is better. Each language self-times its inner loop (reps auto-scaled past a 200 ms floor, so startup is excluded). The fastest cell per bench is green; the ai axis is tinted; a dot means no implementation (or an unavailable toolchain). The net row ranks the columns — the geometric mean of each language's per-bench times, so every bench counts in proportion and no single heavy one dominates (a language is fast at some things, slow at others). Below it, shown but not ranked: bell (bignum — not every language has it) and setup (cold start: source → trivial result, so compiled languages pay their compile).

This is where a tuned C solver pulls away from an interpreted one — the honest other side of the language table above.
