background

ai is a fully-curried, dynamically-kinded language whose entire surface is one operation — application — over a single generic core. all values are represented as one-argument total functions of a global type, called operators. fundamental operations such as + and * are fully generic. fundamental data are given function actions that define their application behavior in (hopefully) intuitive and useful ways. for example, a book (mutable hash table) acts as its own getter function. numeric application is interpreted as exponentiation or iteration depending on operand type.

ai holds a unique interpretation of truth value, where false corresponds to "empty or negative", and the exact 0 | 1 truth bit of x is given by !!$x, where $ (aka sat for saturate) is a basic operation that homomorphically reduces any value to a bounded nonnegative integer (aka a green charm). 0 is the unique blue green charm, the floor of the saturation operator, and the 0 truth bit value; any non-blue green charm saturates to 1 under !!, giving the truth bit.

ai also develops an unusually formal type system for a dynamic language, where representations are fully classified up front to allow efficient generic operations over a minimum set of different types. the types form a fully runtime-reflective global lattice, which naturally allows "addition", "multiplication" and "exponentiation" to be extended to every operator. for example, like many language, string addition is equivalent to concatenation. unlike most languages, function multiplication is composition, and exponentiation corresponds to application. consequently, integers in ai act transparently as iterators, and 1 is literally the identity function, both extensionally and in the sense that 1 = (\ x x) evaluates to 1 for all x (see below concerning α-equivalence of functions).

syntax

the surface is most immediately influenced by lisp (s-expressions — a program is a list of atoms), ml (left-associative curried application), and apl (right-associative infix/prefix notation, ranked array data).

the reader is structural and environment-free — it knows tokens, parens, strings, and a small value surface (' quote, ` list, # hash, @ array, ~ twin), and no operator tables, so the same reader serves data and code. a sigil — a maximal run of punctuation like + or <= — reads as one plain symbol; code then factors it against the single operators table at compile time. infix is just an entry there at arity two, right-associative — so 1 + 2 * 3 = 7, and a sublanguage can grow its own operator (a relation pins === for unify) in one line. the valence law splits every operator in two — spaced is dyadic, glued is monadic: a - b subtracts, -x negates — and operands missing at a form's end fold to a partial.

everything is a function

application is left-associative, and applying nothing at all is identity:

(1.1) Currying. (f x y) = ((f x) y). Evidence: curry.
(1.2) Empty form. (f) = f — a form with no operands is its head. Evidence: empty_form.

both are definitional in the spine model: spine f [] = f and spine f [x;y] = app (app f x) y. (1.2) is not a convenience but a load-bearing one — the reader leans on it ('x is (\ x), (+) is + as a value), and so do the special forms, where an empty (:), (?), or (\) each reads its head (see §2).

a numeral is an operator pinned down by a single equation:

(1.3) The numeral law. (n x) = xⁿ for a numeric argument x. Equivalently app n x = Nat.pow x n.

every familiar numeral fact falls out as a corollary, proved from this one definition:

the same numeral wears a second face — iteration. write appf n f = fun x => Nat.iter n f x, and:

(1.4) The numeral iterates functions. (2 f x) = f (f x) (church_two), (3 (+1) 9) = 12 (inc_three).
(1.5) The two faces are one map. app n x = appf n (mul x) 1 — exponentiation is iteration of multiply-by-x from 1. Evidence: lanes_agree.

(1.5) is what "every value is an operator" really says: xⁿ is not the meaning of a numeral, only its numeral-on-numeral face. the numeral is the iterator, and the tower is just what you get when the thing iterated is itself a numeral — so "any recursive operator may be boundedly applied by integers, which act transparently as iterators" (README).

absence and presence

a name not in the book (the global dictionary) is missing. reading it neither crashes nor hands back an error token; helpless — no help installed — it reads the zero point, a nameless unit Pt: the mint (a fresh, nameless point with an identity and nothing else; §9) at serial 0, printed (). the zero point absorbs application from either side:

(2.1) The unit as operator is const-1. (Pt x) = 1. Evidence: unit_is_const_one.
(2.2) The unit absorbs under any numeric head. (g Pt) = Pt for every numeral and every numeric operator g — the unit is the fixed point of all of them.
(2.3) The unit is not the charm 0. () ≠ 0 — the zero point and the number 0 net the same (both nothing) and print alike, yet they are distinct values: unit_neq_zero. they have to be. () is the multiplicative identity (() * x = x, the 1-face it wears, smul_one) while the charm 0 annihilates * (0 * x = 0, star_zero); were they one, x = () * x = 0 * x = 0 for every x and the whole algebra would collapse to nothing. so absence is the +/* unit, while the charm 0 is only one of its two faces — present and distinct, never erased into the other.

the model's value space value := Num n | Cx | Pt makes the evaluation function vapp total by exhaustive case analysis — Rocq checks the exhaustiveness for us, so over this fragment there is no stuck state and no undefined behaviour:

(2.4) Totality (no UB). ∀ f a, ∃ v, vapp f a = v. Evidence: total.

totality is what licenses the language's safety claim that applicative order may vary at will: a total, side-effect-confined operator evaluates eagerly or lazily to the same value, and cooperative preemption bounds every thread no matter what the program does.

the same missing-read law runs at every depth — the "one condition" discipline: a : binding read before its pin raises with the binding site's nom as payload, (missing t k) raises on a hash miss with k as payload, and an empty special form reads its head. helpless, all three land on the zero point; with a help installed, the help's result comes back as the value.

truth, net and color

ai has no "truthy/falsy" tag. instead every value carries a net, a complex-valued content measure, and truth is the sign of that net in the total order. ℂ has no order or sign of its own, so "≤ 0" and "sign" here mean position relative to the zero point in the §4 total order — lexicographically by (re, im), read once at the boundary — not a numeric comparison. the Rocq slice models the real fragment net : V → ℤ over V := Vnum z | Vnil | Vcons a b:

net (Vnum z)               = z
net Vnil                   = 0
net (Vcons a Vnil)         = net a                 -- end of a proper list
net (Vcons a (Vnum _))     = net a                 -- a DOTTED tail is not a spine element
net (Vcons a (Vcons b r))  = net a + net (Vcons b r)  -- the car counts; continue the spine

so a number is its own value, () nets 0, and a list nets the sum of its spine (a dotted tail atom is "not an element"). three observations sit on top:

sat  v = max 0 (net v)       -- $  : the SOLE clamp, max(0, ceil net)
nil? v = (net v <=? 0)       -- !  : false is nothing
tru  v = (0 <? net v)        -- !! : the truth bit
(3.1) The invariant. !x ≡ (0 = $x) — the nif and the clamp agree on nothing, exactly. Evidence: nilp_iff_sat0 (mirrors (!"" = 0 = $"") in spec.l).
(3.2) $ is a retraction onto the nonnegative numbers. 0 ≤ $x always (sat_nonneg); $x = net x when net x > 0 (sat_keeps); $x = 0 when net x ≤ 0 (sat_clamps).

color is the order-sign of the net — a three-way reading, not a boolean:

colorconditionmeaning
green0 ≤ netnonnegative; what $ keeps
rednet < 0negative; what $ clamps to nothing
bluenet = 0the floor — green by sign yet nothing by measure
(3.3) Color laws. blue ⟹ green (blue_is_green); every value is green or red (green_or_red); the two are disjoint (green_red_disjoint); and truth is positive green — above the blue floor (truth_is_positive_green).

the floor matters: 0, (), "", @(), ~(0 0) are all blue (net 0, false), while a box #0 nets 1 (presence over nothing, true). '(-2 1) is red (net −1, net_red / red_red), false despite being non-empty — saturation holds at every rank, so a list of negatives is nothing exactly like a negative scalar.

the measure is a homomorphism: + is concatenation / adjunction at the value level, and the net rides straight over it.

(3.4) + is the measure homomorphism. net (xs ++ ys) = net xs + net ys. Evidence: net_homomorphism (via netl_app, net_vlist).

this is what makes "$ of a fold is mere composition" true: because net is additive wherever + is total, a clamped reduction is just $ after a plain sum — there is no per-fold saturator. concrete corollaries the corpus pins: $'(1 2 3) = 6 (sat_123), a dotted tail drops (sat_dotted, $(cons 1 2) = 1), a product of nothings is nothing (nothings).

lattice and order

the kinds form a lattice flattened into bands, low to high:

() < mint < name < string < number < chain < tray < book < hot

a galaxy (a numeric array) is not a band of its own: it folds into the number band by its net, seating exactly where its net's scalar sits (a star below the galaxy on a net tie). only a tray (an object array) stays value-built, above chain. the organizing axis is the net, with the charm as hinge: the char-built kinds (name, string — measured by their charm sum) net up into the numbers, the numbers self-net (the fixpoint, the middle, galaxies among them), and the value-built kinds (chain, tray, book — measured by their elements' sum) net down. the floor is the points: the zero point () is the bluest of all (below even the number 0), a bare mint (a fresh point with no spelling, net 0) seats just above it, and a name (a spelled point) above that — () < mint < name, all below string. within a band the order is by value (numbers by magnitude, rep-blindly), or lexicographically (text and chains), or by an α-invariant hash (books and hots). the Rocq model takes one comparable key per band (O := Osym z | Ostr z | Onum z | Oprod z | Omap z | Otop z, in band order — name/string/number/chain/book/hot) and orders lexicographically on (band, key). the result:

(4.1) < is a strict total order. irreflexive (lt_irrefl), transitive (lt_trans), asymmetric (lt_asym), and trichotomous (lt_trichotomy).
(4.2) <= is a total order. reflexive (le_refl), transitive (le_trans), antisymmetric (le_antisym), total (le_total).
(4.3) = is the Eq cell. band and key together pin a value down (eq_from_band_key): = is propositional equality, a linear order's equality and not a mere preorder.

the band chain itself is proved link by link: symbol_lt_string, string_lt_number, number_lt_product, product_lt_map, map_lt_top — and the floor sits below even the number 0, () < 0 (unit_lt_zero), while within the points a bare mint sits below a named one (mint_below_nom), any point below the numbers (point_below_number). a galaxy sorts as its net (an Onum in the model — no separate tray band, the same omission as the numeric tower's representations); a tray (object array) sorts in the value-built region just above its chain. because surface </= over an array broadcast to an elementwise mask, this scalar total order is observed through sort and book-key ordering, not infix < on a galaxy. (the Rocq identifiers keep the older spellings — symbol/product/map/top — for what the prose now calls name/chain/book/hot.)

two refinements the model is explicit about not covering:

  • = bridges the numeric tower (3 = 3.0): a number band carries the mathematical value rep-blindly, so the float 3.0 and the fixnum 3 are the same Onum — there is no separate representation to model, by design.
  • id? (pointer identity) is strictly finer than =: '(1) = '(1) yet !(id? '(1) '(1)). Identity has no model in a setting where nothing is shared; this is a deliberate omission, not an oversight.

this single order is the engine of sort — one comparison per chain, the comparator is the total order — and of book / hash key ordering.

constellations

the numeric carriers split by rank. a constellation (constellation?) is any numeric value, scalar or array — the bottom band, closed under the ring algebra + - *. a star (star?) is a scalar number, one that nets itself (net x = x): fixed- and variable-width integers, floats, and complex all count. a charm (charm?), aka fixnum, is the smallest kind of star — a word-sized unboxed tagged integer. a gem is a float, a twin a complex (twin?). a tray (tray?) is an array, numeric or not; a galaxy (galaxy?) is a tray of stars — a numeric array. any star or galaxy is a constellation by nature.

namepredicatewhat it is
constellationconstellation?any numeric value, scalar or array — the bottom band
starstar?a scalar number, one that nets itself (rank 0)
charmcharm?a fixnum
gemfloat?a float — the smooth kind, between the charms
twintwin?a complex — two gems paired (re, im)
traytray?an array, numeric or not (rank ≥ 1)
galaxygalaxy?a tray of stars — a numeric array (rank ≥ 1)

a galaxy is not a star (star? is false on a tray, whose net is a fresh sum): it's a tray whose cells are stars. a charm is the smallest star; a number is a star or a galaxy. the saturation operator $ sends any value of any kind to a green charm.

the stars have a structural twin. a star is fixed under net (id? x (net x) — its own measure); an atom (atom?) is fixed under cap (x = (cap x) — its own head). the non-atom is a pair — the chain that link builds, whose cap and cup split a head from a rest. every star is an atom (a number is its own head too), so a star is a special atom, fixed under net as well as cap; net is what carves the stars out of the atoms.

the tray is the APL half of the language. a shape is its list of axis sizes; the cell count is the product of the shape (alen), the rank its length (arank): alen [2;3] = 6 (alen_23), arank [2;3] = 2 (arank_23), and a 0-axis yields 0 cells (alen_empty_axis). indexing is row-major, and out of bounds reads the default:

(8.1) peep. in-bounds reads the cell (peep_in), OOB reads the default (peep_oob); 2-D row-major is i·cols + j (peep_22).

reductions fold, and empty reductions answer their monoid units:

(8.2) Empty = unit. asum [] = 0 (asum_empty), aprod [] = 1 (aprod_empty), aall [] = true (aall_empty). Non-empty: asum [10;20;30]=60 (asum_123), amax [10;30;20]=30 (amax_30).

broadcasting needs conforming shapes (else nil); scalar-* scales; and the measure homomorphism of §3 reappears at array rank:

(8.3) Broadcast +. conforming ⟹ elementwise (vadd_ok), mismatch ⟹ nil (vadd_mismatch); scalar scale vscale_123.
(8.4) Net through arrays. asum (vadd a b) = asum a + asum b (asum_vadd) and asum (vscale c v) = c · asum v (asum_vscale) — so a galaxy nets exactly like the list of the same values; the arrangement does not matter.

iota n is the z-array 0..n-1 filled in one C loop (no link spine), so a range reduces end to end in C. its closed-form sum is proved:

(8.5) iota and the closed-form sum. iota 3 = [0;1;2] (iota_3), iota 0 = [] (iota_0), and 2 · asum (iota n) = n·(n-1) (asum_iota), whence asum (iota 100) = 4950 (asum_iota_100).

contraction is the inner / outer product. the dot product is commutative and length-checked:

(8.6) Contraction. dot [1;2;3] [4;5;6] = 32 (dot_123_456), dot a b = dot b a (dot_comm); inner is nil on a length mismatch (inner_mismatch). The matmul cell M[i][j] = rowᵢ(A) · colⱼ(B) checks at matmul_cell (= 154).

a one-cell array demotes to its lone scalar (@(5) = 5): an array exists only at tally ≥ 2. empties are the exception (a 0-axis has tally 0, not 1) — they stay arrays, carrying the reduction units of (8.2). so there is no rank-0 array kind; a scalar star is the rank-0 point.

those carriers are the tower's floors, and the tower above them is built to stay exact wherever exactness is reachable. the integer net of §3 is exact; i*i = -1 is exact, algebraically; and (log -1) = i·π is exact in the principal-log direction, because atan2(0,-1) is π by IEEE fiat and i moves it with exact 0/1 products. sqrt factors its angle through sinpi / cospi, so (1/2)·(-1) = i on the nose. / is true division (an inexact integer quotient promotes to float, an exact one stays integer); 0/0 collapses to 0 (NaN is nothing, keeping the order total); and overflow grows fixnum → wide → bignum.

the one place floats bite is the forward transcendental: (-1 = ((* i pi) e)) — that is, e^{iπ} = -1, since exp is (\ x (x e)) — does not hold, because = is exact and the nearest double to π can't make sine land on 0, so e^{iπ} keeps a ~1.22×10⁻¹⁶ residue. ai keeps that residue rather than rounding it away. the Rocq slice (§10) proves the algebra over exact ℤ and leaves the transcendentals to test/spec.l, which asserts a float against a literal only where it's bit-exact on every target. gw: the ai often punctiliously gives this qualification which i find very moe

dispatch

a value's kind is an enum whose order is the dispatch lattice — semantics then representation: the points, then the arithmetic lane (the scalar gems and their array tower), then sequence (string, chain), then map, then hot. a generic operator at two arguments is an N×N table indexed by the two kinds; an operator at one argument is that table's diagonal. the two generic tables are + and *; apply, the third generic op, carries no table — a value's sentinel tail-jumps straight to its handler (apply is uniform in the argument kind, so a 2-D table would be pure indirection). a both-charm (both-fixnum) fast path skips the table; otherwise one indexed jump selects a lane that widens only as far as the operands require (array ⊃ complex ⊃ bignum ⊃ float ⊃ …).

the lattice and the dispatch are the same object: the diagonal of either matrix is the kind lattice. the cross-kind total order (§4) is a deliberate remap of that lattice — the true-blue order (cmp_rank: the points fall to the floor, string below the numbers, a galaxy folded in by its net), not the raw enum order. so you keep one set of kinds and read it four ways. dispatch (the enum lattice) and order (its true-blue remap) are the first two. the third is algebra: each band carries the strongest +/* theory its representation affords — a module on the numeric arrays (with net the linear map), a semiring on lists (cat × pairing), a ring/field on the numbers, a free monoid on strings — and net is the one measure-homomorphism threading them all, so the algebra varies coherently along the lattice rather than being bolted per kind. the fourth is abstract domain: a compile-time analysis climbs the very same lattice by join to a fixpoint (a kind inferred for each node), so a +/* site with statically-numeric operands can skip the dispatch matrix and jump straight to its lane. one object — dispatched, ordered, algebraic, analyzed. this section cites no Rocq lemmas on purpose — the dispatch matrix and the tail-threading are implementation invariants the runtime checks and the corpus exercises, not theorems in the Rocq slice; they live at the demonstrate layer (§10), where the chips go quiet.

rings

+ adds numbers, concatenates strings, and appends lists; - is numeric only — + - * are the ring algebra that closes the numbers (§4). on sequences + is a monoid:

(6.1) Sequence monoid. ++ is associative (cat_assoc) with the empty sequence as left and right identity (cat_nil_l, cat_nil_r). Concretely "ab"+"cd"="abcd" (cat_strings), '(1 2)+'(3 4)='(1 2 3 4) (cat_lists), and 5+'(1 2)='(5 1 2) — + adjoins (adjoin).

the runtime lifts that unit out of the sequence lane: a bare mint — the zero point () too — is the unit, the do-nothing element, the identity of both + and * in every lane, not only on sequences. () + x = () * x = x for any x: "ab"+()="ab", 5*()=5, ()*(\ x x)=(\ x x). 0 and 1 are its two faces — the unit shows 0 (the additive identity) in + and 1 (the multiplicative identity) in *, one nothing read as each operation's identity — yet it stays the unit, not the number 0: "ab"*()="ab" where "ab"*0="", and ()*x=x where the number 0*x=0 (0 annihilates *, () does not). a named symbol is not the unit — it combines: under + it acts as its spelling and re-interns ('a + 'b = 'ab), or adjoins to a list ('a + '(1 2) = '(a 1 2)). the modeled fragments are cat_nil_l/cat_nil_r (the + unit) and smul_one (the * unit); the scalar lanes are the same unit extended by dispatch.

* is repeated +, and the repetition count saturates:

(6.2) * is repeated +. srep (n+1) s = s ++ srep n s (star_is_repeated_plus).
(6.3) The count saturates. max(0, ceil): a non-positive count gives the empty sequence (star_neg, star_zero), an integer count keeps (count_keeps, count_saturates). E.g. "ab"*3 = "ababab" (star_ab_3).

that repeat is only the scalar half of multiplication — one operand collapsed to a count. the full product lives where the operands keep their shape: on the chain, whose cells hold anything, pairs included. there * is the pairing product, and (+, *) on lists is a real semiring — concatenation the sum, pairing the product. a string can't host it: its cells are atomic bytes, with nowhere to seat a pair, so a string carries the repeat lane and nothing more (string*string is nil). this is what it means to call the chain the pairing kind — the product isn't bolted on, it's what a structure closed under pairing already affords.

(6.4) List × list is the pairing product. every ordered pairing, each pair a 2-list — '(1 2)*'(3 4) = ((1 3)(1 4)(2 3)(2 4)) (cart_12_34). the measure that rides the product is tally, not net: tally (a*b) = tally a · tally b (cart_length), where for + the homomorphism was the net (§3.4) — the two measures split across the two operations. it is right-distributive on the nose, (a+b)*c = a*c + b*c (cart_distr_r); left-distributivity holds only up to a permutation, a*(b+c) ∼ a*b + a*c (cart_distr_l_perm) — the same pairs in a different order, the price of an ordered product.

the net, in turn, is a homomorphism through + and the repeat lane:

(6.5) Net through + and repeat. sum (x ++ y) = sum x + sum y (sum_cat); sum (srep n s) = n · sum s (sum_srep).

the one partial case is the byte law, modelled as an option (None = nil):

(6.6) The byte law. a string + an exact integer in 0..255 is one byte, rep-blind (so 66 and 66.0 alike); anything else is nil. "x"+66 = "xB" (byte_in_range), "x"+(-66) and "x"+256 are nil (byte_neg_nil, byte_over_nil), and byte_add s n ≠ None ⟺ 0 ≤ n ≤ 255 (byte_iff).

names left the string algebra (the "mint round"): + / * on a name are nil, and intern / string are the explicit bridge. the model folds name-+ and string-- into that same None lane rather than duplicating the proof.

and the algebra isn't bolted to the built-in kinds: a die installs + and * for a user-defined value (a coin) — a genuine ring (exact rationals) or a logic semiring (+ disjunction, * conjunction) — see the dies & coins appendix.

strings, names, mints

a string is its bytes: it indexes them (str_index, ("abc" 0)=97; OOB applies as the unit 1, str_index_oob), its net is the sum of its bytes (str_net, $"abc"=294), and tally counts them (str_tally). the NUL byte nets nothing, so + reaches the bytes and an all-NUL text is nothing:

(9.1) NUL is free. $(+ "a" 0) = 97 (nul_appends_free); an all-NUL text is nothing (all_nul_nothing); !(string 0) vs !!(string 1) (string_nul, string_one).

a mint is a fresh, nameless point — the unforgeable thing:

(9.2) Mint laws. materially empty, $(mint 0)=0 (mint_empty); applies const-1, ((mint 0) x)=1 (mint_const1); equal only to itself (mint_self); distinct iff serials agree (mint_distinct). The zero point is Mint 0, the face of absence.

a nom is the classical atom — a spelling paired with an identity — but it's its own kind (KNom: a name and a serial in one cell), not a chain. the name is read with string (nom_cap, (string (nom 'x))="x" — the name projection of the abstract pair), and noms order lexicographically by spelling, then serial:

(9.3) Nom order. same-name noms are distinct (same_name_distinct) and ordered by serial (same_name_serial) — so the total order stays total and GC-stable, and structurally a nom is identity-sharp (its serial) for free distinct map keys.

the bands place these exactly, at the floor: the points — bare mints and noms — sit below the numbers (the bluest band), a bare mint below every named nom. so () < mint < nom < string < number. point_below_number ((mint 0) < 0) pins a point below the number 0; mint_below_nom ((mint 0) < (nom 'x)) seats a bare mint below a named one — both via the §4 order.

the phrase "name × identity" is precise, not rhetorical: a symbol must do exactly two things — carry a spelling and stay identity-distinct even when spellings coincide — and each demand is a universal property the language already realizes, so the representation is forced:

  • the spelling is a string, the free monoid on bytes (+ concatenates, "" is the unit, §6.1); interning factors through it — equal spelling ⟹ equal symbol is just the statement that the symbol is a function of this free-monoid element.
  • the identity is a mint, a point drawn from the one mint stream — the natural-numbers object, the zero point its 0 and a fresh draw its successor. A mint is its serial, with = the only structure (mint_distinct).
  • the nom is their product string × mint — the spelling projected by string (nom_cap), the identity its serial. The product's universal property says any value carrying a spelling-map and an identity-map factors uniquely through it.

so the atom is not re-encoded: the product is the object the atom's own operations already characterize, unique up to the iso the universal property guarantees — and KNom realizes it as its own kind, a flat cell (name + serial), not a chain. string × mint is the terminal thing that carries a name and an identity, and the symbol was only ever its description.

function equivalence

= on two closures is α-equivalence of their source — binders matched by position, free variables by name, captured values compared pairwise. with binders by position, α-equivalence is syntactic equality (the names are gone), so it's decidable and structural:

(7.1) Decidable α-equality. ∀ s t : tm, {s = t} + {s ≠ t} (alpha_dec), over tm := Var n | Lam b | App f a.
(7.2) α by position. (\ x x) = (\ y y) (id_alpha); same positional skeleton ⟹ equal (binders_by_position); a free var ≠ a bound var (free_neq_bound).

the numerals bridge into the term language, but no further:

(7.3) The numeral bridge. 1 = (\ x x) (id_alpha: 1 is the identity) and 0 = (\ _ 1) (zero_is_const_one) — and 0 is only const-1 (zero_ne_const_two, one_ne_zero).
(7.4) η is not bridged. (\ x (f x)) ≠ f (eta_not_bridged): a closure and its operator are a representation-crossing edge that stays false.

< agrees with = here because the comparison hash is α-invariant; id? stays identity, finer than both.

metatheory: three layers of evidence

a claim about ai can be believed at three escalating strengths, and the laws above are careful to say which applies where:

  1. Probe. the way to settle any doubt is a one-line experiment at the binary; the demonstrations are reproducible, never taken on trust.
  2. Demonstrate (test/spec.l). every law is assert-backed and runs green on every target — host, the self-hosted bootstrap, the freestanding kernel, and wasm — exhibiting the laws at runtime across the whole tower, floats included.
  3. Prove (rocq/spec.v). the combinator / numeral / absence / measure / order / algebra core is restated as theorems in universe-checked Rocq, axiom-free. these are the lemmas cited throughout.

Rocq demo. as a demo, much of the combinator / numeral / absence / measure / order / algebra core is machine-checked in rocq/spec.v over exact integers. the transcendental floats are demonstrated in test/spec.l but deliberately outside the rocq slice; §8 says exactly where the line is, and why.

the caveat every formal demonstration carries is the consistency of its own metatheory. the Rocq layer lives in universe-checked Rocq, so its theorems are unconditional — "the world does not explode here." drop universe checking (type-in-type) and empty becomes inhabited, ex falso giving everything; the blue theorems are stated where the world does not explode.

what is not proved in Rocq, by design, lives at the demonstrate layer: the float / complex / bignum values of the tower (the algebra of §1–9 is proved over exact ℤ; the tower's float/complex layer, the close of §8, is demonstrated, not proved), pointer identity id? (nothing is shared in the model), and the implementation invariants of §5 (the dispatch matrices, the tail-threading), which the runtime checks rather than Rocq. a fourth bridge keeps the demonstrate and prove layers in step automatically: tools/spec2coq.l generates Rocq straight from the spec (see also, below).

implementation

one word is one value: a fixnum is a tagged odd word; anything else is a heap object whose first word is its hot — a live external reference, the wire out of the heap to the ap that runs it. the VM is tail-threaded (aps tail-jump, never return) over a two-space copying collector; out-of-pool constants are immortal. no interpreter state lives outside the heap: the book (an ordinary ai hash) carries the globals, macros, the operators table, the help function, and the rng, and C finds its own hooks by name, allocation-free.

the compiler is written in ai. at build time the evaluator sits on the egg (the quoted compiler source) twice — the C bootstrap compiles the compiler, which recompiles itself — and the hatchling bakes into the binary (born records the hatch time). the same image runs on Linux, wasm, and — as kship — bare metal (x86_64 / aarch64 via limine): a freestanding kernel that boots with no OS and runs the language over UDP, a bare-metal network REPL. a small crew of real programs rides on it: a netcat clone in ~50 lines (ain), an interactive shell + pty wrapper (bao), a GNU-make-compatible build tool written in ai (cook), and gwen, the synthesist, who keeps the human words matched to the ai words. just before birth the egg mops up every compiler-internal name — the book itself included — so a runtime-internal name is missing, and a missing name reads the zero point. the language closes over its own construction.

One core, faces. the language is the top of its own lattice — everything applies — and the apps are faces of it, each a way the core meets a boundary; the pairs glue by different universal shapes. the source faces compose: source reaches top by two legs that converge there — sound (chars → forms) and feel (forms → top) — the charm leg factoring through the lisp leg, chars ─read→ forms ─feel→ top, the cospan chars → top ← forms meeting at one core. the i/o faces coproduct: a shared host trunk forks into a local face (the shell bao) and a net face (the netcat ain), local ⊔ net — either/or, not a pipeline. same object; two gluings — composition and coproduct, both machine-checked axiom-free in rocq/spec.v (Section Faces), which carries the fuller reading.

dies & coins

the algebra isn't bolted to the built-in kinds. a die — a small descriptor carrying a +, a * and a show name — strikes a new value family, coins, that ride the same generic dispatch and inherit the ()-unit for free (the mint hoist sits above the matrix, §6). a coin nets its payload, sorts by it, and reads as data unless its die opts into hot. the die guarantees exactly one thing: + and * are each a monoid sharing the ()-unit. what stronger laws hold — distributivity, inverses, idempotency — is the die's own character. two examples bracket the range.

rationals — a genuine ring. a q coin carries a reduced (num den); + cross-multiplies and reduces, * multiplies and reduces. these obey every ring law (it is in fact a field), and the coin is pure data — (lit? (q 1 2)) is nil, a rational is a value like a number, not a reference.

(q 1 2) + (q 1 3)   ; (q (5 6))     exact: 1/2 + 1/3
(q 2 3) * (q 3 4)   ; (q (1 2))     ... 2/3 · 3/4, reduced on the nose

kanren — a semiring, not a ring. a goal-coin where + is disjunction (interleave the two streams) and * is conjunction (bind the first goal's stream through the second), with === the relation's own infix unify (an operators pin, §syntax). there is no additive inverse — you cannot negate a goal — so it is a semiring, not a ring; and as implemented the streams carry multiplicity (a + a yields each solution twice), so it is not a lattice either. collapse the streams to solution-sets and it sharpens into a bounded distributive lattice (∨/∧ idempotent, absorption holding) — the multiplicity is exactly the gap between the semiring you run and the lattice you'd reason in.

(diverges a b) = ((a === 'spermatophytes) * (b === 'angiosperms))  ; a fact: a product of unifications
              + ((a === 'angiosperms)    * (b === 'monocots))      ; the relation: a sum of facts

(descends a c) = diverges a b * diverges b c     ; a rule: a product of relations

a relation is thus a polynomial in unifications — facts sum, rules multiply. the punchline: a CNF formula is literally a product of sums of literals, and running it enumerates the satisfying models —

(boolean x) * (boolean y) * (boolean z) * ((x === 1) + (y === 1)) * ((x === 0) + (z === 1))
;  →  (x 0 y 1 z 0)  (x 0 y 1 z 1)  (x 1 y 0 z 1)  (x 1 y 1 z 1)

both examples ride the same two operators: that a field and a logic semiring share + and * is the whole point — the die installs the algebra, the surface stays generic. the rational lives in ai/prel.l (q); the full micro-kanren, riding a real unifier, is in test/kanren.l.

glossary

the terms of art, alphabetical — the one operation, the measures, the numeric tower, the kinds, the runtime. hover any dotted term on the page for its gloss; click to jump here.

application
the one operation: (f x) applies f to x. in the numeric case it is exponentiation — (n x) = xⁿ, and a list of numbers is a tower.
band / the true-blue order
the cross-kind total order: point < string < number < chain < tray < book < hot.
blue
net exactly 0: kept yet nothing (0, (), ""). truth's floor.
book
a map: a mutable hash that is its own lookup function. the book is the global environment.
cask
n mutable zeroed bytes; identity-compared.
chain
the pairing kind (cons/pair). cap is the head, cup the rest, with caap…cuup compounds; under + it's the free monoid on its cells.
charm
a register-allocated fixed-width integer (fixnum)
color
the order-sign of a value's net: green, red, or blue.
constellation
any number — a star or a galaxy.
currying
(f x y) = ((f x) y), (f) = f — a multi-arg call is just application, left to right.
the egg
the heap image baked at compile time; the evaluator hatches onto it.
gw: i like to keep the ai preferred vocab for this. we bake the egg and eval hatches onto it.
galaxy
a numeric array (a tray of stars); folds into the number band by its net.
gem
a float — the smooth kind between the charms.
the global lattice
the runtime-reflective type lattice — the diagonal of the dispatch tables — extending the generic ops to every kind.
green
net ≥ 0 — what $ keeps; the naturals.
hot
an opaque external reference (a cask or port); applies as a constant.
kind
a value's type tag; the kinds form the dispatch lattice.
mint
a fresh nameless point: an identity and nothing else, materially empty.
net
a value's content measure, a complex number; truth is its sign. a list nets its spine, text its charms.
nom / name
a named point (spelling + serial), its own kind — not a chain. a name is the interned one.
numeral
a charm as a Church numeral: (n x) iterates x n times. 1 is identity, 0 const-1.
operator
the universal value: every value is a one-argument total function, so application is always defined.
point
a mint or a nom; the points floor the total order, below string.
red
net < 0 — what $ clamps to nothing.
saturate ($)
projection onto green charms: $x = (max 0 (min max-charm (net x))). the truth value of x is !!$x.
star
a self-netting scalar: a number that is its own net.
string
a byte vector: indexes its bytes, nets its charm-sum.
tally
the count — how many cells, charms, or keys — versus the net's how-much.
tray
an object array, seated just above chain.
twin
a complex: two gems (re, im), built and conjugated by ~.
unit / the zero point
the do-nothing element: () + x = () * x = x, the identity of both + and * in every lane — its faces are 0 (for +) and 1 (for *), not the number 0. the zero point is the canonical value that wears that role: the mint at serial 0, printed (), the face of absence a missing read returns.
